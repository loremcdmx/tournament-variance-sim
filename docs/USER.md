# Аватар пользователя — Variance Simulator (FF Старт)

Один зафиксированный аватар. Все продуктовые решения проверяются против него.
Если решение противоречит — пересматривать аватара, не размывать продукт.

## Кто

- Игрок программы FF Старт на **старт-этапе**: ранг 15-16, аби $1-3, нулевая
  интуиция про вариацию.
- Не технарь. Может знать EV/ROI/drawdown интуитивно, но не строго.
  **Не знает** distribution, p95, RoR, standard error, sigma-per-tournament.
- **Предположение о prerequisite-тренажёрах = ноль** (программа в разработке).
  Значит Variance Sim должен быть **самодостаточным**: всё необходимое
  объясняем прямо в нём.
- Повторные визиты (аби$5, аби$10) — **не входят в скоуп текущей версии**.
  Делаем для одного уровня, расширим позже если нужно.

## Контекст

- Платформа FF Старт — **обучающая программа от 0 до аби10**, набор тренажёров
  (≥9 сейчас, до 20+ в перспективе). Variance Sim — один из модулей.
- Программа делится покерными мудростями, основами теории вероятностей,
  калькуляцией EV/ROI, банкролл-менеджментом, ICM, форматной стратегией и т.п.
  **Каждый тренажёр отвечает за свой узкий кусок знаний.**
- Variance Sim отвечает **только за «увидеть и почувствовать вариацию»**.
  Не учит банкролл-менеджмент как дисциплину, не объясняет ICM,
  не строит теорию equity — для этого есть другие тренажёры.
- Заходит **по ссылке** (от куратора, из меню платформы, из дискорда).
- Single-visit **на каждом этапе программы**. Один и тот же кандидат может
  вернуться в Variance Sim повторно через 3-6 месяцев на новом уровне (аби$5,
  аби$10) — но это новая сессия, не «дочитал прошлый раз».
- Браузер: desktop или mobile, без предположений. Русский primary.

## Главный вопрос

Не «выдержит ли мой банкролл моё расписание» — это вопрос опытных регов.

Главное: **понять, что такое вариация в MTT, и что это значит для его игры**.

## Три урока, которые должен унести

1. **Даунсвинг — это норма.**
   -20...-30 баев это не слив, это статистика. Просадки такого размера
   встречаются у +ROI игрока многократно за карьеру.

2. **Малая выборка — шум.**
   200-500 турниров не доказывают ROI. Конвергенция требует тысяч.
   Знание «у меня +15% на 200 турнирах» — это сигнал ниже шума.

3. **PKO имеет другую вариацию, чем freezeout.**
   Bounty-элемент перераспределяет эквити в сторону «убийц», увеличивая
   variance относительно фиксированного призового.

## Что НЕ делает аватар

Эти антипаттерны фиксируют скоуп продукта и cut-list:

- **Не вводит своё расписание.** Пресет ранга 15-16 — дефолт по умолчанию.
- **Не сравнивает три расписания.** Это сценарий для опытных, не для обучения.
- **Не настраивает стили линий / цвета / референсные линии.** Дизайн зафиксирован.
- **Не интересуется PrimeDope-сверкой.** Это инструмент разработчика.
- **Не запускает повторно.** Открыл — посмотрел — ушёл. Saved-state ни к чему.
- **Не разбирается в tilt-моделях, ROI-shock-channels, empirical buckets.**
  Это admin-уровень.

## Что должен почувствовать когда закроет вкладку

> «Окей, теперь я понимаю, что вариация в MTT огромна. -25 баев даунсвинг — это
> не катастрофа, это норма. Мой ROI на 200 турнирах — не доказательство, нужно
> играть тысячи. PKO — отдельная история с другой кривой».

Если аватар закрывает вкладку с этими тремя предложениями в голове — продукт
работает. Если закрывает с «много чартов, я ничего не понял» — продукт сломан.

## Связь с другими тренажёрами FF Старт

Программа в разработке. Что покрывают другие модули — пока неизвестно.

**Текущее правило:** Variance Sim самодостаточен. Не предполагаем что кандидат
прошёл что-то заранее. Не ссылаемся на конкретные внешние тренажёры
(их ещё нет или не финализированы).

**Когда программа сформируется**, добавим в финальную карточку «Что изучать
дальше» прицельные ссылки. Сейчас это TODO в Стадии 5 (polish) или после.

## Что НЕ делает Variance Sim

Эти темы покрываются программой в целом, не этим модулем:

- Bankroll-management как дисциплина
- ICM / push-fold / структура поздних блайндов
- Расчёт эквити / equity vs range
- GTO basics, sizing, range building
- Конкретная стратегия PKO / Mystery / BR

Здесь мы только **показываем вариацию** этих форматов; стратегия — не здесь.

## Тех-профиль

- Десктоп или мобильный, без предположений какое чаще.
- Русский primary; английский как fallback / для скриншотов в дискорде.
- Скорость инета — нормальная (платформа предполагает онлайн-обучение).
- Не знаком с разработкой / SQL / графиками распределений.

## Что это значит для решений

| Решение | Почему |
|---|---|
| Дефолтный пресет обязателен | Single-visit, не вводит ничего |
| PKO — primary формат | Самый популярный в реальности |
| Три урока должны быть видны | Иначе уйдёт без вывода |
| Любая фича оправдывается через «помогает ли одному из трёх уроков» | Иначе скоуп размывается |
| Admin-режим — отдельный путь (`?admin=1` или флаг) | Нужен для тебя, не для аватара |
| Mobile должен работать | Платформа предполагает оба |
| Русский UI primary, EN зеркало | Платформа русскоязычная |

## Что меняется при изменении аватара

Если аватар сдвинется (например, «опытные реги для bankroll planning») —
этот документ перепишется первым, потом план, потом код. Не наоборот.
